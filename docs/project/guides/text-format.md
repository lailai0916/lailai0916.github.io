## 通用

### 文件

- 文件应使用 **UTF-8** 编码。
- 文件末尾应以 $1$ 个 **空行** 结束。
- 文件开头应有一段简短的 **内容概述**，例如「本文将介绍……」。
- 对于尚未撰写或未完成的内容，应使用 `[TODO]` 标记。
- Markdown 文件的扩展名应使用 `.md`。

### 文件名

- 文件名应遵循 **kebab-case**（仅包含 **小写字母** 和 **数字**），避免使用大写字母、空格和中文。
- 文件名的语义应使用 **英文单词** 和 **缩写**，避免使用汉语拼音。
- 文件名包含多个部分时，应使用 **连字符**（`-`）分隔，避免使用下划线（`_`），例如 `text-format.md`。

## 空格

- **中文** 和 **英文、数字** 之间应添加空格。
- **中文** 和 **公式、加粗、斜体、链接** 等格式之间应添加空格。
- **数字** 和 **单位** 之间不应添加空格。
- **全角标点** 前后不应添加空格。
- 专有名词应使用官方定义的书写格式，例如「豆瓣FM」。

## 标点

### 全角 & 半角

- 使用全角中文标点。
- 数字使用半角字符。
- 不重复使用标点符号。

### 用法

- 每句话的末尾应使用 **句号**（`。`）、**问号**（`？`）、**感叹号**（`！`）或 **省略号**（`……`）。
- 应正确使用「的」「地」「得」。
- 应正确使用各种不同的 **连接号**：
  - `-`（U+002D，hyphen，连字符）；
  - `–`（U+2013，en dash，短破折号）；
  - `—`（U+2014，em dash，长破折号）。

## 用词

- 专有名词应使用正确的大小写，例如「GitHub」、「iPhone」。
- 使用可以读懂的缩写，不应使用不地道的缩写。
- 名称和术语首次出现时，使用 `中文（原文，缩写）` 的格式，例如「最大公约数（Greatest Common Divisor，GCD）」。
- 减少不必要的 **形容词** 和 **语气词**。
- 应正确使用「的」「地」「得」。
- 应正确使用「他」「她」「它」。

## Markdown

### 缩进

- Markdown 文本应使用 $2$ <kbd>Space</kbd> 缩进。

### 标题

- 每篇文本最多包含 $1$ 个 **一级标题**，其余内容应使用 **二级、三级、四级标题**，避免使用 **五级、六级标题**。

## LaTeX

- 数学公式应使用 $\LaTeX$。
- 公式不应放在 **标题、加粗、斜体、链接** 等格式中。
- 行间公式前后的 **两个美元符号**（`$$`）应 **单独一行**。
