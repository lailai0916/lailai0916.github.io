## 简介

**最小环**（girth）是图中边权和最小的简单环，至少含 3 个顶点。稠密图用 Floyd 在 $O(n^3)$ 内求解，稀疏图可枚举边后跑 Dijkstra。

## Floyd 算法

Floyd 在求全源最短路的同时可以顺带求出最小环。关键性质：在外层循环枚举中间节点 $k$ 之前，$dis_{i,j}$ 表示仅经过编号在 $[1,k)$ 中节点的 $i\to j$ 最短路。因此，在第 $k$ 次外层迭代开始时，对所有满足 $i<k,j<k$ 的点对，

$$
dis_{i,j}+g_{i,k}+g_{k,j}
$$

就是以 $k$ 为最大编号节点的环的长度（$g$ 为原始邻接矩阵）。枚举并取最小值即得最小环。时间复杂度 $O(n^3)$。

## Dijkstra 算法

依次枚举每条边 $(u,v,w)$，临时删去该边后从 $u$ 跑一次 Dijkstra，得到不经过该边的 $u\to v$ 最短路 $d$，则过该边的最小环长度为 $d+w$。时间复杂度 $O(m(n+m)\log n)$，适合稀疏图。

## 实现

以下为 Floyd 求无向图最小环。

```cpp
#include <bits/stdc++.h>
using namespace std;

using ll=long long;
const int N=305;
const ll inf=0x3f3f3f3f3f3f3f3fll;
ll g[N][N],dis[N][N];
int n,m;

ll min_cycle()
{
	ll ans=inf;
	for(int k=1;k<=n;k++)
	{
		for(int i=1;i<k;i++)
		{
			for(int j=1;j<i;j++)
			{
				if(g[i][k]<inf&&g[k][j]<inf&&dis[i][j]<inf)
					ans=min(ans,g[i][k]+g[k][j]+dis[i][j]);
			}
		}
		for(int i=1;i<=n;i++)
		{
			for(int j=1;j<=n;j++)
			{
				if(dis[i][k]<inf&&dis[k][j]<inf)
					dis[i][j]=min(dis[i][j],dis[i][k]+dis[k][j]);
			}
		}
	}
	return ans;
}

int main()
{
	ios::sync_with_stdio(false);
	cin.tie(nullptr);
	cin>>n>>m;
	for(int i=1;i<=n;i++)
	{
		for(int j=1;j<=n;j++)
		{
			g[i][j]=dis[i][j]=(i==j?0:inf);
		}
	}
	while(m--)
	{
		int u,v;ll w;
		cin>>u>>v>>w;
		g[u][v]=g[v][u]=min(g[u][v],w);
		dis[u][v]=dis[v][u]=min(dis[u][v],w);
	}
	ll ans=min_cycle();
	if(ans==inf)cout<<"No solution.\n";
	else cout<<ans<<'\n';
	return 0;
}
```
